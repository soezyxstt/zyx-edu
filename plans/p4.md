## Backend spec
1. `attempt_feedback`: `attemptId`, `questionIndex` (int, index into snapshot), `payload` (json), unique pair. Payload: `{whyWrong, misconceptionName?, correctApproach: string[] (max 3), reviewHref}`.
2. Feedback worker: for one submission, collect all wrong answers; per question check KV `feedback:{questionId}:{sha256(normalized student answer)}`; for misses, ONE batched Gemini call (JSON array out) covering all uncached mistakes; store rows + KV put each (TTL 30 days). Charges 1 budget unit per submission regardless of mistake count.
3. Attempt summary (deterministic, no AI, computed inline at submit): map questions to concepts via tags/KO links; `strongAreas` = concepts with all correct, `weakAreas` = concepts with any wrong; `recommendedNextSteps` = up to 3 items reusing P2 builders scoped to `weakAreas`. Persist on the attempt row (json column, add if absent).
4. `interventions`: `id`, `studentId`, `courseId`, `conceptName`, `reason` (text), `status` (`active|dismissed|resolved`), `payload` (json `{moduleHref, quizTemplateIds, flashcardCount}`), `createdAt`, `resolvedAt`. Partial unique: one `active` per `(studentId, conceptName)`.
5. Rules in recompute worker. Rule 1 (works from P1A data): >= 3 consecutive failing quiz events for a concept. Rule 2 (auto-activates once P1B trend exists): mastery < 30 AND trend `declining`. Creation skipped when an active row exists. Resolution: mastery > 60 flips to `resolved`. Reason strings exact: `3 consecutive quiz failures`, `low and declining mastery`.

## Frontend spec
Screen A: attempt review (rework).
- Header: score `font-heading text-h3 font-bold` + "/100" `text-body-sm text-muted-foreground`, below `text-body-sm text-muted-foreground` "{correct} of {total} correct".
- Strong/weak: two columns `grid grid-cols-2 gap-8 mt-6` (mobile 1 col). Each: label `text-body-sm font-semibold uppercase tracking-wide text-muted-foreground` ("Strong" / "Needs work"), then plain list `space-y-1 text-body-base`; weak items prefixed `TrendingDown size-3.5 text-status-error` inline. NO cards.
- Questions: `divide-y divide-border mt-8`. Row `py-4`: question number + prompt (existing rendering with `MathText`), student answer line, correct answer line (`text-body-sm`, correct value in `text-status-success`).
- Wrong questions only: collapsible "Why this was wrong" trigger `Button variant="ghost" size="sm"` with `ChevronDown size-4` (rotates 180deg, `transition-transform duration-200`). Content (`animate-in fade-in duration-200`): misconception name as `h6 text-body-sm font-semibold` when present; `whyWrong` paragraph `text-body-base`; `correctApproach` as `ol list-decimal pl-4 space-y-1`; link "Review this concept" `text-primary text-body-sm hover:underline` to `reviewHref`.
- Feedback pending (worker not finished): in place of the collapsible, `text-body-sm text-muted-foreground` with inline `Loader2 animate-spin size-3.5` "Generating explanation". Poll with react-query every 3 s, max 30 s, then Retry per UI-STD error row.
- Next steps: bottom section reusing the P2 checklist row styling, heading "Recommended next steps".

Screen B: intervention banner (dashboard, above Today).
- Container: `border-l-4 border-status-warning bg-status-warning/10 rounded-md px-4 py-3`. The ONLY allowed filled container on the dashboard.
- Row 1: `TriangleAlert size-4 text-status-warning` + `text-body-base font-medium` "Struggling with {conceptName}" + right `text-body-sm text-muted-foreground` "{reason}".
- Row 2 (`mt-2`): 3 compact links `flex flex-wrap gap-4 text-body-sm text-primary`: "Review module", "Practice quiz", "{n} flashcards".
- Row 3 right-aligned: `Button variant="ghost" size="sm"` "Dismiss" (calls dismiss endpoint, banner exits `animate-out fade-out duration-200` then removal).
- Multiple active interventions: render only the newest, plus `text-body-sm text-muted-foreground` "+{n} more" linking to the concept map.
