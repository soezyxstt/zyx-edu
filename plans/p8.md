# P8: Vectorize Migration (CONDITIONAL, may never run)

Objective is reliable retrieval, not Cloudflare purity. Pinecone behind the P3 adapter is the default forever. Open this phase ONLY if a trigger fires, AND the size gate passes. "Never migrate" is a fine outcome.

Depends on: P3 adapter. Read `globals.md` first.

## Activation gate (check first, do nothing until one trigger fires)
Triggers (any one):
- [ ] Pinecone storage near 2 GB
- [ ] Sustained retrieval p95 > 500 ms
- [ ] Quality regression or unfavorable Pinecone free-tier change

Size gate (must ALSO pass):
- [ ] Vectorize free tier holds about 6,500 vectors at 768 dims and about 39k topK queries/mo. Count Pinecone records first. Over budget: stay, or evaluate paid options.
- [ ] Decision recorded in `docs/baselines.md` (the quarterly ritual lives in P10).

## Progress (only if activated)
- [ ] Trigger + counts documented BEFORE any code
- [ ] `workers/vector-api/` deployed
- [ ] `VectorizeStore` adapter done
- [ ] Dual-write soak done
- [ ] Read cutover + 1 week soak done
- [ ] Pinecone removed
- [ ] Gate green

## If activated: build steps
1. Create `workers/vector-api/` (Vectorize has no public REST data plane): `POST /query|/upsert|/delete`, shared-secret header, bound index. `wrangler vectorize create zyx-edu --dimensions=768 --metric=cosine`.
2. Create `VectorizeStore` implementing the P3 `lib/vector-store.ts` interface (course namespaces map to Vectorize namespaces).
3. Env `VECTOR_STORE=pinecone|vectorize|dual`. Dual: the Inngest sync worker writes both, reads stay Pinecone.
4. Backfill by EXPORTING vectors from Pinecone (no re-embedding, saves Gemini quota); re-embed via `embedTexts` only if export fails.
5. Shadow-compare 50 real queries (log top-5 ids from both stores) before read cutover; 1 week soak; then remove Pinecone writes, delete index, remove keys.

## If activated: out of code

- [ ] Monitor Vectorize query-dimension usage weekly during soak.
- [ ] After soak: delete the Pinecone index, remove keys.
