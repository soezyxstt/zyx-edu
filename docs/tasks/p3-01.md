# Task P3-01: Vector Store Adapter Integration

## Context
As part of Phase 3 (Grounded AI Tutor), we must abstract our vector indexing client to allow conditional migrations or testing of other databases (e.g. Cloudflare Vectorize) in the future.

## Objectives
- Implement `lib/vector-store.ts`.
- Expose a `VectorStore` interface with `query`, `upsert`, and `delete` methods.
- Expose a `PineconeStore` class implementing `VectorStore` wrapping `lib/pinecone.ts`.
- Migrate all current call-sites of `lib/pinecone.ts` (e.g., question-generation retrieval in `lib/question-generator.ts` or `lib/generation-pipeline.ts`) to use the new adapter interface.

## Verification
- Lint and build pass (`npm run lint && npm run build`).
- Verify query behavior still routes correctly to Pinecone index.
