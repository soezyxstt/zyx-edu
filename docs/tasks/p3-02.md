# Task P3-02: KV Cache Client with Rate Limiting

## Context
Tutor query repeats should serve from Cache at zero Gemini cost. We use Cloudflare Workers KV as the storage layer.

## Objectives
- Create `lib/kv-cache.ts`.
- Implement Cloudflare KV REST client endpoints: `kvGet`, `kvPut(key, value, ttlSeconds)`, `kvDelete`.
- Enforce a 1.5-second timeout on requests (failure must result in a silent cache miss).
- Implement a daily write counter stored under `meta:writes:{yyyy-mm-dd}`. Stop writes once the count reaches 900 to safeguard the free limit (1,000 writes/day).

## Verification
- Test GET/PUT/DELETE commands with KV credentials in `.env`.
- Check timeout and write counter logic.
