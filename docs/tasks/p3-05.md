# Task P3-05: Chapter Summary Generator Endpoint

## Context
Students need to generate on-demand summary sheets for chapters.

## Objectives
- Create the backend handler `app/api/student/summarize-chapter/route.ts`.
- Implement a cache-first logic checking KV cache for `summary:{chapterId}:{materialVersion}`.
- Gemini Flash prompt returning key concepts, common mistakes, important formulas, and practice recommendations.
- Keep execution profile-free (cached summaries can be shared safely among all students).

## Verification
- Test GET request on the route to ensure it serves cached summaries correctly.
