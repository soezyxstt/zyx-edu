# Task P3-04: Grounded Tutor RAG Pipeline Core

## Context
Builds the 3-tier RAG pipeline that answers user questions grounded in course materials.

## Objectives
- Create `lib/tutor-rag.ts` implementing the core 3-tier pipeline:
  - **Tier 1 (Cacheable)**: KV cache lookup. Miss -> Pinecone search namespace -> fetch chunks -> Gemini Flash structured output: `{answer: string, sources: Array, matchedConcepts: string[], confidence: number}`. Cache entry if confidence >= 0.6.
  - **Tier 2 (Deterministic Addendum)**: Check if `matchedConcepts` contains any `profile.weakConcepts`. Attach personal review links/due flashcard count templates.
  - **Tier 3 (Personalized LLM)**: If matched concepts are in `weakConcepts` and budget allows, fire a personalized Gemini session adjusting explanation strategy.
- Update `tutor_session_summaries` and append learning event rows after each chat turn.
- Map the tutor drawer endpoint (`/api/student/tutor`) to use `tutor-rag.ts`.

## Verification
- Mock/test cache hits, misses, and personalization triggers.
