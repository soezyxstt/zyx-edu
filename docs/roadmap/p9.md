## Backend spec
1. DO state: `{sessionState: 'lobby'|'question'|'reveal'|'ended', currentQuestionIndex, deadlineTs, participants: {userId, name, score, connected}, answersByQuestion}`. Persist to DO storage on every transition (restart mid-session recovers).
2. Protocol: client `join {token}`, `answer {questionIndex, choice, clientTs}`; server `lobby_update`, `question {prompt, options, deadline}` (correct answer NEVER sent before reveal), `reveal {correctIndex, perOptionCounts}`, `leaderboard {top10, yourRank}`, `ended {finalBoard}`, `state_sync` on rejoin.
3. Scoring: base + linear time bonus from SERVER receive time. `clientTs` is diagnostics only. Rate limit: 1 answer per question per user. Cap 150 participants per room.
4. On end: POST results to callback; rows into the 2 tables; one `learning_events` row per answer (mastery updates from live play too).

## Frontend spec
Host console: lobby = join code centered `font-heading text-h1 tracking-widest` (the one allowed centered layout, it is a projector screen) + participant count live `tabular-nums`; controls row `flex gap-2`: Start / Next / Reveal / End as `Button` (`default`, `outline`, `outline`, `destructive`). During question: per-option horizontal count bars (`h-3 rounded-md bg-muted` track, `bg-primary` fill, `transition-[width] duration-300`).

Student view: join input (6-char, uppercase, `Input` `tracking-widest text-center`); question fullscreen: countdown `font-heading text-h5 tabular-nums` top-right turning `text-status-error` under 5 s; options reuse quiz-player styles; lock after answer (`opacity-60 pointer-events-none`); reveal: correct option `border-status-success bg-status-success/10`, your wrong pick `border-status-error`; leaderboard between questions: top 10 divider rows, rank `tabular-nums`, own row accented `border-l-2 border-primary`, row reorders with `transition-transform duration-300`. Connection dot top-right: `size-2 rounded-full` `bg-status-success` / `bg-status-warning` (reconnecting pulse `animate-pulse`).
