# P3: Grounded AI Tutor (RAG + Citations + Learner Memory + KV Cache)

Goal: tutor answers from course materials with citations, knows the learner (adapts to weak concepts), and serves repeats from cache at zero Gemini cost.

Flag: `FEATURE_TUTOR_RAG=1` (off: drawer behaves exactly as today). Depends on: P0, P1A. Read `globals.md` first.

Pipeline: question > learner profile > KV cache > (miss) vector search > context > Gemini > answer (+ deterministic personal addendum).

## Backend spec
1. Cache keys: tutor `tutor:{courseId}:{chapterId|all}:{sha256(normalized question)}` (normalize: trim, lowercase, collapse whitespace, strip punctuation). Summary `summary:{chapterId}:{materialVersion}`.
2. Tier 1 (cacheable): KV get. Hit: return stored `{answer, sources, matchedConcepts}` instantly, log `cache:hit` row to `ai_usage_events` with 0 tokens, do NOT charge budget. Miss: embed question, `vectorStore.query` topK 6 in `course_{courseId}` namespace (chapter filter when provided), hydrate chunks/KOs from DB, prompt WITHOUT any student data, Gemini 2.5 Flash JSON mode. Output contract: `{answer: string(markdown), sources: [{type: 'chapter'|'ko'|'question', id, label}], matchedConcepts: string[], confidence: number}`. KV put TTL 7 days only when confidence >= 0.6 and write counter < 900.
3. Retrieval empty (all similarity < 0.5): answer generally, set `grounded: false`, empty sources, never invent citations.
4. Tier 2 (deterministic, per student, per request, zero AI): if `matchedConcepts` intersects `profile.weakConcepts`, attach `addendum: {conceptName, message, reviewHref, flashcardsDue}` built by string template. Runs on hits AND misses.
5. Tier 3 (personalized AI, uncached): only when the question's matched concept is in `weakConcepts` AND daily budget allows. Second Gemini call with profile injected (instructions: smaller steps, check prerequisite, address the student's recent mistake pattern). Charged to budget. Budget exhausted: serve Tier 1+2 silently.
6. After each exchange: deterministic update of `tutor_session_summaries` (append matchedConcepts, bump count). Also 1 `learning_events` row, eventType `tutor_question`.
7. Summary endpoint: same cache-first shape, fixed prompt returning `{keyConcepts: string[], commonMistakes: string[], importantFormulas: string[], practiceRecommendations: string[]}`, TTL 30 days, key embeds material version so edits invalidate naturally. Profile-free by design.
8. AI Gateway cache ON (out-of-code) as a free second layer.

## Frontend spec
Screen A: tutor drawer (edit `components/course/tutor-drawer.tsx`).
- Keep existing Sheet chrome, header, input row. Changes are inside the assistant message block only.
- Sources footer (when `sources.length > 0`): below answer text, `mt-3 pt-2 border-t border-border`. Label "Sources" `text-body-sm font-medium text-muted-foreground`. Then inline links `flex flex-wrap gap-2`: each `text-body-sm text-primary hover:underline` with `BookOpen size-3.5` (chapter) or `FileQuestion size-3.5` (question) inline before label. Chapter links go to the material page, question links to the review page.
- Ungrounded notice (when `grounded === false`): line above answer, `text-body-sm text-muted-foreground italic`, copy: "Not covered in your course materials. General explanation:".
- Personal addendum (when present): block after sources, `border-l-2 border-primary bg-muted rounded-md px-3 py-2 mt-3`, enter `animate-in fade-in slide-in-from-bottom-2 duration-300`. Line 1: `Lightbulb size-4 text-primary` inline + `text-body-sm font-medium` "Builds on {conceptName}". Line 2 `text-body-sm text-muted-foreground`: "You scored {score} here recently." Line 3: two links `text-body-sm text-primary hover:underline`: "Review chapter" and "{n} flashcards due".
- Cache-hit render: skip any typing/streaming affordance, paint instantly.
- States: send error per UI-STD error row inside the thread; budget exhausted: muted row "Daily limit reached. Resets at midnight."

Screen B: Summarize this chapter.
- Trigger: `Button variant="outline" size="sm"` with `Sparkles size-4`, label "Summarize", right-aligned in the material viewer header row.
- Opens `Sheet side="right"` width `sm:max-w-md`. Title: chapter name `text-h6 font-heading`.
- Body: 4 sections in order: Key concepts, Common mistakes, Important formulas, Practice. Each: `h3 text-body-sm font-semibold uppercase tracking-wide text-muted-foreground mb-2`, then bullet list `list-disc pl-4 space-y-1 text-body-base` (formulas via `MathText`). `space-y-6` between sections. No cards.
- Loading: 4 skeleton groups (label bar `w-24 h-3` + three `h-4` lines). Button shows inline `Loader2` while open-and-loading.
