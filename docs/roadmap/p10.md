## Frontend spec
Ops page: six sections, dividers, typographic stats per UI-STD 2.3. Each section: title + status word `text-body-sm font-semibold` colored `text-status-success|warning|error` ("healthy", "degraded", "down") + 2 to 4 figures + "checked {relative}" `text-muted-foreground`. Thresholds: Gemini > 200/day warning, > 240 error; KV writes > 700 warning, > 900 error; any Inngest failure 24 h warning; sync queue depth > 100 warning. Auto-refresh via react-query every 60 s with `RefreshCw` spin during fetch. No charts on v1 of this page.
