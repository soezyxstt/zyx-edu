### `🔴 CRITICAL MISMATCH — assessmentObjects`

**Problem:** The Bundle V1.1 marks `pattern`, `reasoningType`, `estimatedSteps`, `applicationLevel` as **optional** with default values (`"general"`, `"analytical"`, `1`, `1`).

**The database schema has these columns as `notNull()` with NO DEFAULT value.**

| DB column | `.notNull()` | Has default in DB? |
|---|---|---|
| `pattern` | Yes | **No** |
| `reasoningType` | Yes | **No** |
| `estimatedSteps` | Yes | **No** |
| `applicationLevel` | Yes | **No** |

**Impact:** If the bundle omits these fields, an `INSERT` will fail with a
`NOT NULL constraint failed` error. The importer MUST supply values for all
four columns, even though the bundle marks them optional.

## Summary

### Fields that map directly (✅) — 40 fields

All content fields (titles, descriptions, markdown, question types, enum
values, difficulty, tags, options, front/back) map one-to-one between the
bundle and the database. The bundle structure correctly mirrors the schema.

### Fields requiring transformation (⚡) — 22 instances

| Transformation | Tables affected |
|---|---|
| UUID generation for PKs | All 11 tables |
| FK resolution (`$ref`, `ko$ref`) | KOs, flashcards, knowledgeRelationships |
| Nesting → FK inference (courseId, chapterId, conceptId, sourceId, setId) | chapters, KOs, websiteMaterials, flashcardSets, flashcards, assessmentSources, assessmentObjects |
| Array index → order field | chapters, KOs, assessmentObjects |
| author email → user UUID | masterTeachingDocuments, assessmentSources |
| markdown `ref="..."` → `koId="<uuid>"` | websiteMaterials.canonicalMarkdown |
| SHA-256 hash computation | websiteMaterials, flashcardSets, assessmentSources, assessmentObjects |
| slug auto-generation (absent → derived from title) | websiteMaterials |
| `source` block → `metadata._source` JSON sub-key | knowledgeObjects, flashcards |
| `source.file` → `originalFilename` | assessmentSources |

Every DB column without a bundle counterpart has either:
- A schema default (`DEFAULT value`) → importer omits it, Drizzle fills it

## Recommended Minimum Changes Before Implementing `scripts/import-bundle.ts`

### Must fix (blocking)

1. **`assessmentObjects` four required columns** — The Bundle V1.1 marks
   `pattern`, `reasoningType`, `estimatedSteps`, `applicationLevel` as
   optional with documented defaults. The DB has `NOT NULL` with no defaults.
   **Fix:** Either (a) alter the DB schema to add defaults to these four
   columns, or (b) document in the importer that it MUST supply these values
   when the bundle omits them. The V1.1 proposal's documented defaults
   (`"general"`, `"analytical"`, `1`, `1`) are acceptable for the importer to
   inject, but the mismatch between the proposal and the actual schema must be
   resolved one way or the other.

2. **`assessmentSourceChapters` population** — The bundle has assessment
   sources at the course level with no chapter-link information. The DB table
   `assessmentSourceChapters` links sources to chapters, and the existing app
   reads this for chapter-scoped queries. **Fix:** The importer should parse
   the YAML frontmatter from `sourceMarkdown`, extract the `chapters:` list,
   match chapter names to bundle chapters, and insert rows into
   `assessmentSourceChapters`. Without this, assessment sources are orphaned
   from chapters.

### Should fix (non-blocking, but recommended before v1 ships)

3. **Source metadata storage** — Five bundle locations carry `source`
   metadata that has no DB column. The most impactful gaps are on
   `websiteMaterials` and `assessmentSources` (where tracking origin is most
   useful). **Fix:** Use `structuredContent` on `websiteMaterials` to store
   `_source`. For `assessmentSources`, consider adding a nullable JSON
   `metadata` column (consistent with the pattern used by `knowledgeObjects`,
   `flashcards`, etc.). For `courses` and `chapters`, append to `description`
   as a fallback.

4. **`origin` field on `assessmentSources`** — When importing from a bundle,
   the `origin` should be `"generated"` rather than the default `"uploaded"`.
   **Fix:** The importer should set `origin: "generated"` explicitly instead
   of relying on the DB default.

5. **User role for bundle author** — The bundle `author` block only has
   `email` and `name`. If the user is created (not found), the DB defaults
   `role` to `"student"`. Bundle authors typically need `"admin"` or
   `"teacher"` for the content-upload operations. **Fix:** Add an optional
   `role` field to the `author` block, or have the importer always create
   bundle authors with role `"teacher"`.

### Nice to have (deferrable)

6. **`technicalStandardTerm` override** — If bilingual courses are common,
   add an optional `technicalStandardTerm` field to the localization block
   in the bundle. Currently trivial (default `"id"` works for ID-first
   content).

7. **`assessmentObjectConcepts` / `assessmentObjectKos` population** —
   Deferred to a future Bundle V2. The current V1.1 is content-focused and
   these join tables are optimization for RAG retrieval, not content
   authoring.
