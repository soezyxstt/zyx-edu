# Purpose

This document defines the canonical authoring contract used by external AI workspaces (Google AI Studio, ChatGPT, Claude, etc.) and programmatic bundle generators to produce course content for Zyx Academy.

This version (BSD v2.1 Final) is frozen and serves as the single source of truth for the Zyx ingestion pipeline.

---

# Global Rules and Constraints

## 1. Branding and Typography

* **Brand Name**: Always write as **Zyx** (capitalized first letter, lowercase "yx"). Never write in all-caps except in literal file paths.
* **No Em or En Dashes**: Never use em dashes (—) or en dashes (–) in any content, titles, or descriptions. Use commas, semicolons, hyphens, or periods instead.
* **Math Notation**: Use standard LaTeX formatted inline with single dollar signs (`$ ... $`) and display block equations with double dollar signs (`$$ ... $$`). Do not use brackets like `\( ... \)` or `\[ ... \]`.

---

## 2. Forbidden Fields

Do not attempt to generate or include database-managed fields:
* Autogenerated primary keys (UUIDs) and foreign keys (except bundle-local `$id` references)
* System timestamps (`createdAt`, `updatedAt`)
* Internal hashes (`generationHash`, `sourceHash`, `canonicalQuestionHash`)
* Vector indices (`vectorId`, `pineconeVectorId`)
* Student progress, mastery levels, review intervals, or recommendation logs

---

# Bundle Assembly Structure

A Course Bundle is a single JSON document. It must match the following nested parent-child layout:

```json
{
  "metadata": {
    "schemaVersion": "1.1.1",
    "generatedAt": "2026-06-18T10:00:00Z",
    "generatedBy": "gemini-workspace/fisika-1a",
    "language": "id",
    "courseCode": "FI1101"
  },
  "author": {
    "email": "admin@zyx.id",
    "name": "System Admin"
  },
  "course": {
    "title": "Fisika Dasar 1A",
    "category": "Sains",
    "description": "Mata kuliah fisika dasar semester 1.",
    "chapters": [
      {
        "$id": "ch-kinematics",
        "title": "Kinematika Satu Dimensi",
        "description": "Mempelajari gerak lurus dalam satu dimensi.",
        "concepts": [],
        "knowledgeObjects": [],
        "websiteMaterials": [],
        "flashcardSets": []
      }
    ],
    "knowledgeRelationships": [],
    "assessmentSources": []
  }
}
```

# Entity Specifications

## 1. Metadata Block (Bundle Level)
Top-level metadata block containing parsing directives and audit trails.

* **Nesting**: Bundle Root.
* **Properties**:
  * `schemaVersion` (string, **Required**): Must be exactly `"1.1.1"`.
  * `generatedAt` (string, Optional): ISO-8601 timestamp of bundle generation.
  * `generatedBy` (string, Optional): Name of the AI agent, system, or operator producing the bundle.
  * `language` (string, Optional): Primary course language (e.g. `"id"`, `"en"`).
  * `courseCode` (string, Optional): Institutional course code (e.g. `"FI1101"`).

---

## 2. Author Block (Bundle Level)
Provides account attribution for content ownership logs.

* **Nesting**: Bundle Root.
* **Properties**:
  * `email` (string, **Required** if author block is present): Email of the bundle author. Used by the importer to look up or associate the system user.
  * `name` (string, Optional): Full name of the author.

---

## 3. Course (Course Level)
Represents the root learning course.

* **Nesting**: Bundle Root (`course` object).
* **Properties**:
  * `title` (string, **Required**): The official course catalog title.
  * `category` (string, **Required**): Broad discipline category (e.g. `"Sains"`, `"Matematika"`, `"Rekayasa"`).
  * `description` (string, Optional): A high-level description of the syllabus.

---

## 4. Chapter
A logical partition representing a major syllabus unit. **1 Chapter = 1 Master Teaching Document (MTD) in the database.**

* **Nesting**: Nested under `course.chapters[]`.
* **Properties**:
  * `$id` (string, **Required** if referenced by assessment sources): A bundle-local identifier (e.g. `"ch-kinematics"`).
  * `title` (string, **Required**): Title of the chapter.
  * `description` (string, Optional): Summary of chapter contents.
* **Ordering**: Derived automatically from the array index position of the chapter (1-based order index).
* **Recommended Volume**:
  * 5 to 15 concepts per chapter.
  * 15 to 30 Knowledge Objects per chapter.
  * 20 to 40 flashcards per chapter.

---

## 5. Concept
An abstract conceptual milestone within the course syllabus.

* **Nesting**: Nested under `chapter.concepts[]`.
* **Properties**:
  * `$id` (string, **Required**): A bundle-local identifier used to link Knowledge Objects.
  * `canonicalSlug` (string, **Required**): Unique lowercase kebab-case slug representing the concept (must be unique across the entire course).
  * `localizations` (array, **Required**): Localized display names and search terms.
* **Localization Block Properties**:
  * `lang` (string, **Required**): Must be `"id"` (Indonesian) or `"en"` (English).
  * `displayName` (string, **Required**): The human-readable name of the concept.
  * `aliases` (array of strings, Optional): Synonyms and search phrases to assist RAG semantic mapping.
* **Scope**: Nesting under `chapters[]` in the bundle is for localized reference resolution during ingestion. In the database, concepts are global to the course. No direct concept-to-chapter table exists; mappings are computed dynamically through KOs referencing the concept.

---

## 6. Knowledge Object (KO)
Atomic educational unit. These are individual retrieval nodes for tutoring.

* **Nesting**: Nested under `chapter.knowledgeObjects[]`.
* **Properties**:
  * `$id` (string, **Required**): A bundle-local identifier used to link flashcards and relationships.
  * `title` (string, **Required**): The topic title of the KO.
  * `concept$ref` (string, **Required**): Refers to the `$id` of a concept declared in the same chapter.
  * `conceptName` (string, Optional): Fallback concept display name string for backward compatibility.
  * `content` (string, **Required**): Markdown text of the KO. Must be self-contained (define all variable symbols inline; do not reference preceding text).
  * `type` (string, **Required**): Must be one of the following enums:
    * `concept_overview` (introduces the concept in plain terms)
    * `definition` (formal academic definition)
    * `formula` (equations and parameters)
    * `example` (worked application problems)
    * `misconception` (common student errors)
    * `exercise` (inline practice problems)
    * `summary` (bulleted topic recaps)
    * `objective` (formal learning objectives)
  * `bloomLevel` (string, **Required**): Must be one of: `remember`, `understand`, `apply`, `analyze`, `evaluate`, `create`.
  * `difficulty` (string, Optional): Must be one of: `easy`, `medium`, `hard`. Defaults to `medium`.
  * `importance` (string, Optional): Must be one of: `high`, `medium`, `low`. Defaults to `medium`.
  * `tags` (array of strings, Optional): Category index tags.
  * `source` (object, Optional): Optional `SourceInfo` tracking metadata.
* **Ordering**: Mapped to the `learningOrder` column. Derived automatically from the KO's array position index (1-based index) inside the `knowledgeObjects` list.

---

## 7. Knowledge Relationship
Declares graph dependency edges between KOs.

* **Nesting**: Flat array under `course.knowledgeRelationships[]`.
* **Properties**:
  * `sourceKo$ref` (string, **Required**): The `$id` of the source KO.
  * `targetKo$ref` (string, **Required**): The `$id` of the target KO.
  * `type` (string, **Required**): Must match one of the following enums:
    * `prerequisite` (source KO is required before learning target KO)
    * `related` (general conceptual overlap)
    * `extends` (target KO builds directly on source KO)
    * `example_of` (source KO is a worked example of target KO)
    * `misconception_of` (source KO is a misconception mapping to target KO)

---

## 8. Website Material
Reading text for the web interface.

* **Nesting**: Nested under `chapter.websiteMaterials[]`.
* **Properties**:
  * `title` (string, **Required**): Title of the reading material.
  * `slug` (string, Optional): URL slug. If omitted, the importer derives it from the title.
  * `canonicalMarkdown` (string, **Required**): Main text containing custom triple-colon compiler blocks.

### Compiler Custom Containers
To link KOs into website materials, use custom container blocks referencing the KO `$id` via the `ref` attribute:

```markdown
:::concept {ref="ko-displacement-def"}
Perpindahan didefinisikan sebagai...
:::
```

Supported block tags:
* `:::learning_objective {bloomLevel="..."}`
* `:::concept {ref="..."}`
* `:::definition {ref="..."}`
* `:::formula {ref="..."}` (variable tables must be written inside using markdown tables)
* `:::formula_reference {linkedFormulaBlockId="..."}`
* `:::engineering_insight {discipline="..."}`
* `:::example {ref="..."}` (must contain **Problem** and **Solution** headings)
* `:::misconception {ref="..."}` (must contain **Misconception** and **Correction** headings)
* `:::exercise`
* `:::summary`
* `:::warning {collapsible="true"}` or `:::note {collapsible="true"}`
* `:::glossary_term {term="..."}`
* `:::chart` (visual plot details parsed as YAML)
* `:::graph` (mathematical desmos plot details parsed as YAML)
* `:::diagram` / `:::flowchart` (visual node structure parsing)

### Visual Block Specifications

When using visual blocks within `canonicalMarkdown`, the compiler parses and validates the structures according to the following schemas:

#### 1. Diagrams & Flowcharts (`:::diagram` or `:::flowchart`)
*   **Edge Parsing**: Define connections using standard `-->` arrows. Valid formats:
    *   `Source --> Target`
    *   `Source -- Label --> Target`
    *   `Source --> |Label| Target`
*   **Automatic Node Extraction**: Node IDs are automatically generated from edge names by converting them to lowercase and replacing non-alphanumeric characters with hyphens.
*   **Step Normalization**: Prefix node names with `"Step {Number}: "` or `"Langkah {Number}: "` to let the parser extract step/phase numbers.
*   **Constraints**:
    *   `diagram`: Must contain at least **1 node** and **1 edge**.
    *   `flowchart`: Must contain at least **2 nodes**.

#### 2. Graphs (`:::graph`)
*   **Syntax**: Written in YAML.
*   **Keys**:
    *   `functions`: Array of mathematical equations (e.g. `functions: ["y = sin(x)"]` or `equation: "y = 2*x + 1"`).
    *   `domain`: Axis limits (e.g. `domain: { min: -10, max: 10 }` or `domain: [-10, 10]`).
    *   `samples`: Density of sample calculation points (defaults to `200`).
*   **Constraints**: Must contain at least **1 plottable mathematical function**.

#### 3. Charts (`:::chart`)
*   **Syntax**: Written in YAML.
*   **Keys**:
    *   `chartType`: Must be one of `"line"`, `"bar"`, `"scatter"`, or `"pie"`.
    *   `xLabel` / `yLabel`: Optional axis titles.
    *   `data`: Array of data point objects.
*   **Constraints**: Must contain at least **1 data point**.

---

## 9. Flashcard Set
Groups flashcards together.

* **Nesting**: Nested under `chapter.flashcardSets[]`.
* **Properties**:
  * `title` (string, **Required**): Title of the flashcard set.
  * `flashcards` (array, **Required**): The list of flashcards.

---

## 10. Flashcard
Review questions targeting atomic active recall.

* **Nesting**: Nested under `flashcardSet.flashcards[]`.
* **Properties**:
  * `ko$ref` (string, **Required**): Refers to the `$id` of the KO containing the material being tested.
  * `front` (string, **Required**): Question prompt. Must test exactly one atomic fact.
  * `back` (string, **Required**): The correct answer.
  * `explanation` (string, Optional): Explains the reasoning behind the answer.

---

## 11. Assessment Source
Represents exam papers (UTS, UAS, Tryouts, Quizzes).

* **Nesting**: Flat array under `course.assessmentSources[]`.
* **Properties**:
  * `title` (string, **Required**): Title of the exam paper.
  * `category` (string, **Required**): Must be exactly one of: `"quiz"`, `"tutorial"`, `"uts"` (midterm), `"uas"` (final), `"tryout"`.
  * `year` (integer, **Required**): The academic year (e.g. `2026`).
  * `semester` (integer, Optional): Academic semester (e.g. `1` or `2`).
  * `chapters` (array of strings, Optional): Mapped chapter `$id`s or titles.
  * `sourceMarkdown` (string, **Required**): The raw text of the exam paper containing all question descriptions.
  * `assessmentObjects` (array, **Required**): The list of individual questions.

---

## 12. Assessment Object
An individual question indexing the exam paper.

* **Nesting**: Nested under `assessmentSource.assessmentObjects[]`.
* **Properties**:
  * `questionType` (string, **Required**): Must be `"multiple_choice"` or `"short_essay"`.
  * `difficulty` (integer, **Required**): Rating scale from `1` to `10` (enforced by the validator as an integer).
  * `options` (array of strings, **Required** for multiple choice): Choice strings prefixed with letters (e.g., `["A. 10 m", "B. 20 m"]`).
  * `questionMarkdown` (string, **Required**): The text of the question.
  * `answerMarkdown` (string, **Required**): Step-by-step mathematical or conceptual explanation leading to the correct answer.
  * `pattern` (string, **Required**): Must match one of the allowed engine patterns:
    * `"direct_computation"`
    * `"graph_interpretation"`
    * `"proof"`
    * `"parameter_analysis"`
    * `"modeling"`
  * `reasoningType` (string, **Required**): Must be one of: `"procedural"`, `"conceptual"`, `"analytical"`.
  * `estimatedSteps` (integer, **Required**): Steps required to solve (typically `1` to `4`).
  * `applicationLevel` (integer, **Required**): Typically `1` or `2` based on bloom level depth.
